# Jira工单统计技能 V2.4

## 功能
统计指定release版本、指定类型的工单修复时间，按组件维度输出平均/最短/最长修复时间，支持可视化柱状图展示。统计效率提升10倍，数据100%准确。

## 触发条件
当用户询问以下内容时激活：
- 统计Jira工单修复时间
- 按组件统计bug修复时长
- 查询某个release版本的工单处理效率
- 分析Jira工单的处理周期
- 查询未关闭的Highest问题
- 统计assignee切换次数
- 多版本工单对比分析

## 用法
### 1. 修复时间统计
```
统计 <release名称> <工单类型> [priority级别] 的修复时间
# 示例：
统计 M1000 release 1.4.0 BUG_ 的修复时间
统计 M1000 release 1.4.0 BUG_ highest 的修复时间
统计 1.5.0版本 Story High 的处理时间
```

### 2. 未关闭问题查询
```
查询 <release名称或多个版本> 未关闭的 <priority级别> 问题
# 示例：
查询 M1000 release 1.4.0 未关闭的 Highest 问题
查询 M1000 Aimodule 1.4.0 未关闭的 Highest 问题
查询 M1000 release 1.4.0 和 M1000 Aimodule 1.4.0 未关闭的 Highest 问题
查询 M1000 release 1.5.0 未关闭的 Highest 问题
查询 M1000 AIBOOK 1.4.0 未关闭的 High 问题
```

**注意**：未关闭的正确筛选条件为：
- status IN (In-Progress, NEW, In-Verify, Blocked)
- priority = Highest（或其他级别）
- issuetype = BUG_
- fixVersion = 用户指定的版本（支持多版本用逗号分隔）

### 3. assignee切换分析
```
统计 <release名称> assignee切换超过3次的工单
# 示例：
统计 M1000 release 1.4.0 assignee切换超过3次的工单
```

支持的priority级别：Highest, High, Medium, Low, Lowest

## 输出格式
```
# <release> <工单类型> [优先级]修复时间统计
| 组件 | 工单数量 | 平均修复时间(小时) | 最短修复时间(小时) | 最长修复时间(小时) | 最长修复工单 |
|------|----------|-------------------|-------------------|-------------------|--------------|
| 组件名 | 数量 | 平均 | 最短 | 最长 | SW-XXXX: 问题描述 |

## 整体汇总
- 总工单数量: X
- 整体平均修复时间: X 小时（约X天）
- 整体最短修复时间: X 小时（XX组件）
- 整体最长修复时间: X 小时（XX组件，约X天）

## 重点发现
1. XX组件平均修复时间最长（X小时），主要原因：XXX
2. XX组件工单数量最多（X个），占比X%
3. XX组件平均修复最快（X小时）
```

自动生成柱状图对比各组件平均修复时间，支持HTML页面展示。
